## 一、测试概述

### 1.1 测试目的

本文档基于开题报告中定义的 6 大功能模块，制定功能测试用例，验证系统是否满足设计需求。测试覆盖所有用户角色（管理员、商家、消费者）的核心操作路径和异常场景。

### 1.2 测试范围

| 模块编号 | 模块名称 | 对应开题报告章节 |
|---------|---------|--------------|
| M1 | 用户管理模块 | 主要内容第1点 |
| M2 | 商品分类与管理模块 | 主要内容第2点 |
| M3 | 购物车与订单模块 | 主要内容第3点 |
| M4 | 库存管理模块 | 主要内容第4点 |
| M5 | 销售统计可视化模块 | 主要内容第5点 |
| M6 | 保健科普信息模块 | 主要内容第6点 |

### 1.3 测试环境

## 八、综合与集成测试用例

| 编号 | 用例名称 | 测试步骤 | 预期结果 |
|------|---------|---------|---------|
| T-01 | 完整购物流程 | 消费者登录 → 浏览商品 → 加入购物车 → 结算 → 填写收货信息 → 提交订单 → 自动支付 → 查看订单 | 全流程顺畅，订单状态正确，库存已扣减，购物车已清空 |
| T-02 | 订单生命周期 | 下单(0) → 支付(1) → 发货(2) → 确认收货(3) | 每一步状态正确流转，操作按钮随状态变化 |
| T-03 | 取消订单完整链路 | 下单 → 取消 → 检查库存 → 检查日志 | 订单变为已取消，库存回滚，日志记录取消回滚 |
| T-04 | 多角色协作 | admin 创建商品 → user01 购买 → shop01 发货 → user01 确认收货 | 三个角色协作完成完整订单流程 |
| T-05 | 数据一致性 | 下单后检查：订单金额 = Σ(单价×数量)，库存变动 = 订单数量 | 金额和库存数据完全一致 |
| T-06 | 页面刷新保持登录 | 登录后刷新页面 | 保持登录状态（Token 存在 localStorage） |
| T-07 | 退出登录 | 点击退出 | 清除 Token，跳转首页，购物车/订单不可访问 |
