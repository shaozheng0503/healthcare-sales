## 二、数据库 ER 关系图

### 2.1 实体关系（文字描述，可用 draw.io 绘制）

```
┌─────────┐     1:N     ┌──────────┐     N:1     ┌──────────┐
│  user    │────────────▶│  orders   │◀────────────│ order_   │
│          │             │           │             │ item     │
│ id (PK)  │             │ id (PK)   │             │ id (PK)  │
│ username │             │ order_no  │             │ order_id │
│ password │             │ user_id   │             │product_id│
│ role     │             │ status    │             │ price    │
│ status   │             │ pay_amount│             │ quantity │
└─────┬────┘             └──────────┘             └────┬─────┘
      │                                                │
      │  1:N                                     N:1   │
      ▼                                                ▼
┌─────────┐                                    ┌──────────┐
│  cart    │─────────────────────N:1───────────▶│ product  │
│          │                                    │          │
│ id (PK)  │                                    │ id (PK)  │
│ user_id  │                                    │category_id│
│product_id│                                    │ name     │
│ quantity │                                    │ price    │
└──────────┘                                    │ stock    │
                                                └────┬─────┘
      ┌─────────┐                                    │
      │ article  │          ┌──────────┐        N:1  │
      │          │          │stock_log │◀────────────┘
      │ id (PK)  │          │          │
      │ title    │          │ id (PK)  │        ┌──────────┐
      │author_id │          │product_id│        │ category │
      │ status   │          │ type     │        │          │
      │view_count│          │change_qty│        │ id (PK)  │
      └──────────┘          │order_no  │        │ name     │
                            └──────────┘        │parent_id │
                                                └──────────┘
```

### 2.2 关系说明

| 关系 | 说明 |
|------|------|
| user → orders | 一个用户可以有多个订单（1:N） |
| user → cart | 一个用户可以有多条购物车记录（1:N） |
| user → article | 一个作者可以发布多篇文章（1:N） |
| orders → order_item | 一个订单包含多个商品明细（1:N） |
| product → order_item | 一个商品可以出现在多个订单明细中（1:N） |
| product → cart | 一个商品可以被多个用户加入购物车（1:N） |
| product → stock_log | 一个商品可以有多条库存变动记录（1:N） |
| category → product | 一个分类下可以有多个商品（1:N） |
| category → category | 分类自关联，parent_id 实现二级分类（自引用） |

## 五、订单状态机图

```
                    ┌──────────┐
         下单创建──▶│ 待支付(0) │
                    └────┬─────┘
                         │
                ┌────────┼────────┐
                ▼                 ▼
         ┌──────────┐     ┌──────────┐
    支付 │ 已支付(1) │     │ 已取消(4) │
         └────┬─────┘     └──────────┘
              │                 ▲
              ▼                 │
         ┌──────────┐          │
    发货 │ 已发货(2) │    取消订单
         └────┬─────┘   (退回库存)
              │
              ▼
         ┌──────────┐
  确认收货│ 已完成(3) │
         └──────────┘


状态流转规则：
  待支付(0) ──支付──▶ 已支付(1)
  待支付(0) ──取消──▶ 已取消(4)
  已支付(1) ──发货──▶ 已发货(2)
  已支付(1) ──取消──▶ 已取消(4) + 退回库存
  已发货(2) ──确认──▶ 已完成(3)
```

## 六、技术架构层次图

| 层次 | 技术 | 职责 |
|------|------|------|
| **展示层** | Vue 3 + Element Plus | 页面渲染、用户交互、数据展示 |
| **路由层** | Vue Router 4 | 路由管理、权限守卫、页面跳转 |
| **状态层** | Pinia | 全局状态管理（用户登录信息） |
| **通信层** | Axios | HTTP 请求封装、请求/响应拦截器 |
| **可视化** | ECharts 5 | 图表渲染（折线图/饼图/柱状图） |
| **接入层** | Spring MVC | REST API 接收、参数校验、统一响应 |
| **认证层** | JWT + Interceptor | 令牌认证、用户上下文注入、权限校验 |
| **业务层** | Spring Service | 核心业务逻辑、事务管理 |
| **持久层** | MyBatis-Plus | ORM 映射、CRUD 简化、分页、自定义 SQL |
| **数据层** | MySQL 8.0 + Druid | 关系型数据存储、连接池管理 |

## 八、数据库表清单

| 序号 | 表名 | 中文名 | 字段数 | 主要用途 |
|------|------|--------|--------|---------|
| 1 | user | 用户表 | 11 | 存储三类用户（管理员/商家/消费者）的账号信息 |
| 2 | category | 商品分类表 | 8 | 二级分类树，通过 parent_id 自关联实现 |
| 3 | product | 商品表 | 15 | 商品信息、价格、库存、预警值、销量 |
| 4 | cart | 购物车表 | 6 | 用户-商品关联，联合唯一索引防重复 |
| 5 | orders | 订单表 | 14 | 订单主信息、状态流转、收货信息 |
| 6 | order_item | 订单详情表 | 9 | 订单商品明细，存储价格和名称快照 |
| 7 | stock_log | 库存变动日志 | 11 | 记录每次库存变动，支持审计追溯 |
| 8 | article | 科普文章表 | 10 | 文章内容、状态管理、浏览量统计 |
