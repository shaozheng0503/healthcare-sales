# 部署说明

## 一、环境准备

### 1.1 必须软件

| 软件 | 版本要求 | 验证命令 | 下载地址 |
|------|---------|---------|---------|
| Java (JDK) | 11 或更高 | `java -version` | adoptium.net |
| MySQL | 8.0 或更高 | `mysql --version` | dev.mysql.com |
| Node.js | 16 或更高 | `node -v` | nodejs.org |

### 1.2 可选软件

| 软件 | 用途 |
|------|------|
| IntelliJ IDEA | 后端开发和运行 |
| VS Code | 前端开发 |
| Navicat / DBeaver | 数据库管理 |

---

## 二、数据库初始化

### 2.1 创建数据库

打开 MySQL 命令行或图形化工具，执行建库脚本：

```bash
mysql -u root -p < database/healthcare_sales.sql
```

或在 Navicat 中：
1. 新建连接 → 连接到 MySQL
2. 右键 → 运行 SQL 文件 → 选择 `database/healthcare_sales.sql`
3. 点击「开始」执行

### 2.2 验证数据库

```sql
USE healthcare_sales;
SHOW TABLES;
```

应显示 8 张表：
```
article, cart, category, order_item, orders, product, stock_log, user
```

验证初始数据：
```sql
SELECT username, role FROM user;
-- 应有 6 条记录

SELECT COUNT(*) FROM product;
-- 应返回 12

SELECT COUNT(*) FROM orders;
-- 应返回 8
```

## 三、后端部署

### 3.1 修改配置

编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/healthcare_sales?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
    username: root
    password: 你的MySQL密码    # ← 修改这里
```

### 3.2 启动方式一：IDEA 运行

1. 用 IDEA 打开 `backend` 目录（作为 Maven 项目导入）
2. 等待 Maven 下载依赖（首次约 3-5 分钟）
3. 找到 `HealthcareSalesApplication.java`
4. 右键 → Run

### 3.3 启动方式二：命令行

```bash
cd backend

# Linux / macOS
chmod +x mvnw
./mvnw spring-boot:run

# Windows
mvnw.cmd spring-boot:run
```

首次运行会自动下载 Maven 和项目依赖。

### 3.4 启动方式三：打包为 JAR

```bash
cd backend
./mvnw clean package -DskipTests
java -jar target/healthcare-sales-1.0.0.jar
```

### 3.5 验证后端启动

看到以下日志说明启动成功：

```
Started HealthcareSalesApplication in X.XXX seconds
```

测试接口：
```bash
curl http://localhost:8080/api/category/tree
# 应返回 JSON 分类数据
```

## 四、前端部署

### 4.1 安装依赖

```bash
cd frontend
npm install
```

### 4.2 开发模式运行

```bash
npm run dev
```

访问 http://localhost:3000

### 4.3 生产构建

```bash
npm run build
```

产物在 `frontend/dist/` 目录，可部署到 Nginx 等静态服务器。

### 4.4 Nginx 部署示例（可选）

```nginx
server {
    listen 80;
    server_name localhost;

    # 前端静态资源
    location / {
        root /path/to/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 五、完整启动步骤（快速参考）

```bash
# 1. 初始化数据库
mysql -u root -p < database/healthcare_sales.sql

# 2. 修改数据库密码
vim backend/src/main/resources/application.yml

# 3. 启动后端（新开终端）
cd backend && ./mvnw spring-boot:run

# 4. 启动前端（新开终端）
cd frontend && npm install && npm run dev

# 5. 打开浏览器
# 消费者前台: http://localhost:3000
# 管理后台:   http://localhost:3000/admin
# 账号密码:   admin / admin123
```

## 七、常见问题排查

### 7.1 数据库连接失败

```
Communications link failure
```

**解决**：
1. 确认 MySQL 服务已启动：`mysql -u root -p` 能否连接
2. 确认 `application.yml` 中的密码正确
3. 确认数据库 `healthcare_sales` 已创建

### 7.2 后端启动端口被占用

```
Port 8080 already in use
```

**解决**：
```bash
# 查找占用进程
lsof -i :8080    # macOS/Linux
netstat -ano | findstr 8080   # Windows

# 结束进程或修改端口
# application.yml 中修改 server.port
```

### 7.3 前端 npm install 慢

**解决**：使用国内镜像
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### 7.4 Maven 下载慢

**解决**：修改 Maven 镜像为阿里云，在 `~/.m2/settings.xml` 中添加：
```xml
<mirrors>
  <mirror>
    <id>aliyun</id>
    <mirrorOf>central</mirrorOf>
    <url>https://maven.aliyun.com/repository/central</url>
  </mirror>
</mirrors>
```

### 7.5 Java 版本不兼容

```
Unsupported class file major version
```

**解决**：确认使用 Java 11+，不要用 Java 8。
```bash
java -version
# 应显示 11.x 或更高版本
```
