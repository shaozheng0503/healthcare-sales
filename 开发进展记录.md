## 已完成工作

### ✅ 阶段一：数据库设计（2026-03-18）

**文件**：`database/healthcare_sales.sql`

| 表名 | 说明 |
|------|------|
| `user` | 用户表（管理员/商家/消费者三种角色） |
| `category` | 商品分类表（支持二级分类，parent_id=0为顶级） |
| `product` | 商品表（含库存预警字段） |
| `cart` | 购物车表（user_id+product_id联合唯一） |
| `orders` | 订单表（5种状态：待支付→已支付→已发货→已完成/已取消） |
| `order_item` | 订单详情表（商品快照，防止改价影响历史订单） |
| `stock_log` | 库存变动日志表（入库/出库/手动调整/取消回滚） |
| `article` | 保健科普文章表 |

**设计要点**：
- `order_item` 存储商品名称/价格快照，订单历史不受商品改价影响
- `stock_log` 完整记录每次库存变动，支持追溯
- `product` 有 `stock_warning` 字段，支持低库存预警
- 所有时间字段使用 `DATETIME` + `ON UPDATE CURRENT_TIMESTAMP`

## 快速启动步骤

```bash
# 1. 执行数据库
mysql -u root -p < database/healthcare_sales.sql

# 2. 修改数据库密码
# 编辑 backend/src/main/resources/application.yml 第10行 password

# 3. 启动后端（Java 11）
cd backend
mvn clean package -DskipTests
java -jar target/healthcare-sales-1.0.0.jar

# 4. 启动前端
cd frontend
npm install
npm run dev
# 访问 http://localhost:3000

# 测试账号
# admin / admin123 → 管理员后台
# shop01 / admin123 → 商家后台
# user01 / admin123 → 消费者前台
```

> ⚠️ 注意：初始 SQL 中的 BCrypt 密码值需要用代码生成正确值后替换，
> 可以在测试类中运行：`new BCryptPasswordEncoder().encode("admin123")`
