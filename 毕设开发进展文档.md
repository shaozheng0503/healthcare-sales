## 一、项目概述

本系统是一套面向中小型保健用品销售企业的 Web 管理系统，覆盖**管理员、商家（员工）、消费者**三类用户角色，实现商品管理、库存管理、订单处理、销售统计和保健科普等核心功能。

**项目目标**：

- 替代传统人工管理，减少数据错误，提高企业运营效率
- 实现商品信息、库存数据、订单数据的统一信息化管理
- 通过可视化数据分析为企业决策提供支持
- 改善消费者购物体验

### 3.2 分层设计

采用标准三层架构（Controller → Service → Mapper）：

| 层次 | 职责 |
|------|------|
| **Controller** | 接收HTTP请求，参数校验，调用Service，返回统一Result |
| **Service** | 核心业务逻辑，事务管理，跨Mapper调用 |
| **Mapper** | 数据访问，简单CRUD用MyBatis-Plus，复杂SQL用XML |
| **Entity** | 数据库表映射实体类 |
| **Interceptor** | JWT认证，权限校验，用户上下文注入 |

### 3.3 前后端分离

- 前端：`http://localhost:3000`（Vite开发服务器）
- 后端：`http://localhost:8080/api`（Spring Boot）
- 通信：JSON格式，通过 `Authorization: Bearer {token}` 传递JWT令牌
- 跨域：后端配置全局CORS，开发阶段允许所有来源

#### 商品表（product）

#### 购物车表（cart）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT UNSIGNED | PK | 主键 |
| user_id | BIGINT UNSIGNED | NOT NULL | 用户ID |
| product_id | BIGINT UNSIGNED | NOT NULL | 商品ID |
| quantity | INT | NOT NULL, DEFAULT 1 | 购买数量 |
| created_at / updated_at | DATETIME | | 时间戳 |

**联合唯一索引**：`uk_user_product(user_id, product_id)`，同一用户同一商品只有一条记录，重复加购时更新数量。

#### 订单表（orders）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT UNSIGNED | PK | 主键 |
| order_no | VARCHAR(30) | NOT NULL, UNIQUE | **订单编号**（格式：HC+时间戳+4位随机数）|
| user_id | BIGINT UNSIGNED | NOT NULL | 下单用户ID |
| total_amount | DECIMAL(10,2) | NOT NULL | 订单总金额 |
| pay_amount | DECIMAL(10,2) | NOT NULL | 实付金额 |
| status | TINYINT | NOT NULL, DEFAULT 0 | **0**待支付 **1**已支付 **2**已发货 **3**已完成 **4**已取消 |
| pay_type | TINYINT | | 支付方式：1=模拟支付 |
| pay_time | DATETIME | | 支付时间 |
| receiver_name | VARCHAR(50) | NOT NULL | 收货人姓名 |
| receiver_phone | VARCHAR(20) | NOT NULL | 收货人手机号 |
| receiver_address | VARCHAR(255) | NOT NULL | 收货地址 |
| remark | VARCHAR(500) | | 买家备注 |
| created_at / updated_at | DATETIME | | 时间戳 |

**订单状态流转**：

```
下单 → [待支付(0)] → 支付 → [已支付(1)] → 发货 → [已发货(2)] → 确认收货 → [已完成(3)]
                  ↓                    ↓
              取消订单              取消订单
                  ↓                    ↓
             [已取消(4)]           [已取消(4)]（退回库存）
```

---

#### 订单详情表（order_item）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT UNSIGNED | 主键 |
| order_id | BIGINT UNSIGNED | 关联订单ID |
| order_no | VARCHAR(30) | 订单编号（冗余存储，便于查询）|
| product_id | BIGINT UNSIGNED | 关联商品ID |
| product_name | VARCHAR(100) | **商品名称快照**（防止商品改名影响历史）|
| product_image | VARCHAR(255) | 商品图片快照 |
| price | DECIMAL(10,2) | **购买时单价快照**（防止改价影响历史）|
| quantity | INT | 购买数量 |
| total_price | DECIMAL(10,2) | 小计（price × quantity）|

**设计要点**：订单详情存储商品名称和价格的**快照**，确保历史订单数据不受商品后续修改影响。

**设计说明**：完整记录每一次库存变动，支持库存追溯和审计，是库存管理的核心辅助表。

### 6.3 核心业务逻辑实现情况

#### ✅ JWT认证流程

```
客户端请求 → AuthInterceptor.preHandle()
  ├── 读取 Authorization: Bearer <token>
  ├── JwtUtil.parseToken(token) 解析
  ├── 提取 userId + role → UserContext.set()
  └── 请求通过，进入Controller

Controller/Service中随时可调用：
  UserContext.getUserId()  → 当前用户ID
  UserContext.getRole()    → 当前用户角色
  UserContext.isAdmin()    → 是否管理员

请求结束后：
  afterCompletion() → UserContext.clear()  清理ThreadLocal防止内存泄漏
```

#### ✅ 下单防超卖（乐观锁）

```sql
-- 使用数据库原子操作，条件扣减库存
UPDATE product
SET stock = stock - #{quantity},
    sales_count = sales_count + #{quantity}
WHERE id = #{productId}
  AND stock >= #{quantity}   -- 关键：只有库存足够才更新
```

```java
int affected = productMapper.deductStock(productId, quantity);
if (affected == 0) {
    throw BusinessException.of("商品库存不足（并发）：" + product.getName());
}
```

#### ✅ 订单创建事务

```
@Transactional
createOrder():
  for each item:
    1. 查询商品（校验状态/库存）
    2. deductStock（原子扣减）
    3. 插入 stock_log（记录出库）
    4. 删除对应购物车记录
  → 插入 orders 记录
  → 批量插入 order_item 记录

  任何步骤失败 → 整体回滚
```

#### ✅ 取消订单退回库存

```
cancel():
  查询所有 order_item
  for each item:
    rollbackStock（回滚库存）
    插入 stock_log（记录取消回滚，type=4）
  更新 orders.status = 4（已取消）
```

  // 需要登录但未登录 → 跳转登录页

    next({ name: 'Login', query: { redirect: to.fullPath } })
    return
  }
  // 角色不匹配 → 跳转首页
  if (to.meta.roles && !to.meta.roles.includes(userStore.userInfo?.role)) {
    next({ path: '/' })
    return
  }
  next()
